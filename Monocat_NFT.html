  • flow.core.fact_events: This table contains information about different events on the Flow blockchain. In this query, it is used to retrieve distinct transaction IDs (tx_id) for mint events related to Monocat NFTs.

  • flow.core.fact_transactions: This table stores transaction-level data on the Flow blockchain. The main query operates on this table to calculate various metrics based on the mint transactions. It includes filtering based on transaction ID (tx_id) obtained from the "mint_tx" CTE and the successful transaction status (tx_succeeded).

These tables, fact_events and fact_transactions, are part of the Flow blockchain's core data schema and are typically utilized to extract relevant information for analysis and reporting purposes.
